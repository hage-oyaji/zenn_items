## いま、AIの勢力図はこうなっている

2026 年 5 月初頭の AI 界隈を一言で言うと、**「層が分化し、契約が組み替わり、エージェントが永続化しつつある時期」**です。基盤モデルは Claude / OpenAI / Google AI の三者がそれぞれ別の方向を向き、コーディングツールは Cursor と GitHub Copilot が同じ基盤を使いながらまったく異なる体験を組み立てています。OpenAI は ChatGPT 9 億 WAU の規模と Microsoft / AWS / 自社直の三経路を確保し、Claude は Pragmatic Engineer の "most loved" 46% と JetBrains CSAT 91% を背景にシニア層を押さえ、Google AI は Gemini 3.1 Pro が $2/$12(≤200K)という「フロンティア級で最安」の価格帯でコスパとモダリティ幅で攻めています。コーディングツール側は GitHub Copilot が IDE 網羅度と FedRAMP / LTS で公的機関を、Cursor が SDK 公開と GPT-5.5 / Opus 4.7 の即日採用で前衛を担うという棲み分けが鮮明です。

この俯瞰を支える論点を 3 つに絞ると、次のようになります。

- **マルチクラウド時代の幕開け** — Microsoft - OpenAI 提携改定(2026-04-27、$135B / 約 27% 出資、Azure 独占解消)と OpenAI on AWS Bedrock 限定プレビュー(2026-04-28)が 24 時間以内に同期し、API 顧客は OpenAI direct + Azure(非独占)+ AWS の 3 経路から調達できる構図に変わりました。
- **コーディングツールの SDK 化** — Cursor SDK Public Beta(2026-04-29、`npm install @cursor/sdk`、Local/Cloud/Self-hosted、Rippling / Notion / Faire / C3 AI 採用)で「IDE 単体製品」から「Agent Platform + SDK」へ位置付けが移り、Futurum Group は「Agent Execution Runtime」と表現しました。
- **課金モデルの再設計** — GitHub Copilot は Usage-Based Billing(2026-06-01 発効、AI Credits 1 credit = $0.01)へ移行し、コード補完と Next Edit Suggestions は無制限のまま、Chat / Agent / Cloud Agent / Code Review などの高負荷機能だけを実消費に振り分けます。

## 基盤モデル3社のポートレート

## コーディングツール2強のポートレート

Cursor と GitHub Copilot は、基盤モデルを採用する IDE / アシスタント層に位置するため、同じ Opus 4.7 や GPT-5.5 を使っていてもプロダクトとしての性格は大きく違います。Cursor は「最新モデルを最速で取り込んで、書く速度を再定義する」方向に振っていて、Copilot は「GitHub プラットフォームに深く食い込み、コア体験を壊さずに課金を組み替える」方向で守りと攻めを両立しようとしています。

### Cursor ─ IDE をやめて Agent Execution Runtime に着替えた前衛

Cursor のここ数週間の動きは「IDE という枠から逃げる」一連の流れでした。2025-10-29 の 2.0 から始まり、2026-04-02 の 3.0、04-13 の 3.1、04-24 の 3.2 と 4 月内だけで 3 連続のメジャー更新を投入したうえで、2026-04-29 には Cursor SDK の TypeScript Public Beta を公開しました。`npm install @cursor/sdk` で導入でき、Local / Cloud / Self-hosted の 3 ランタイム、token-based pricing、codebase indexing / semantic search / MCP servers / skills / hooks / subagents / Resumable Cloud Sessions、Composer 2 を含む全モデルが揃っています。先行採用は Rippling / Notion / Faire / C3 AI と SaaS 4 社が並び、Futurum Group は同日に「Cursor 3.2 Reframes the IDE as an Agent Execution Runtime」と表現する評価記事を公開しました。翌 2026-04-30 にはエージェントハーネス研究記事が出て、CursorBench / Keep Rate / Tool reliability 2-3 9s / モデル別ハーネス調整(OpenAI patch-based / Anthropic string replacement)という独自の品質測定ロジックを開示しました。一方、同じ 2026-04-30 の forum では Composer 2 の Linux 32GB OOM・token usage 10x(47 件 reply の最多議論)・chat crash・extension host 不安定が同時多発しており、攻めの裏側で運用負荷の歪みも見えています。

## 直近の注目トピック

短期間に多くのことが起きていますが、5 月以降の地形を変えそうな動きを 4 件に絞って深掘りします。

### マルチクラウド時代の幕開け ─ 24 時間で書き換わった調達経路

2026-04-27 に Microsoft - OpenAI 提携が改定され、7 年続いた Azure 独占供給契約が解消されました。Microsoft の出資は $135B / 約 27%(as-converted diluted basis)に積み上がり、IP license は 2032 まで継続するものの非独占化、Revenue share の 20% は 2030 まで継続だが新規上限が設けられ、AGI escape clause も削除されました。OpenAI 製品は引き続き「Azure first ship」ですが、独占ではなくなった意味は大きいと見ています。翌 2026-04-28 には OpenAI on AWS Bedrock 限定プレビュー(GPT-5.5 / GPT-5.4 / Codex on Bedrock / Bedrock Managed Agents (powered by OpenAI)、IAM / PrivateLink / guardrails / CloudTrail 継承)が発表され、API 顧客は OpenAI direct + Azure(非独占)+ AWS の 3 経路から選べるようになりました。エンタープライズ調達の RFP テンプレートは、ここから書き直しが入ると見ています。

### Cursor SDK と「Agent Execution Runtime」 ─ IDE の解体と再構築

Cursor SDK の TypeScript Public Beta(2026-04-29)は、Cursor の自己定義を「IDE 単体製品」から「Agent Platform + SDK」へ書き換えました。Local / Cloud / Self-hosted の 3 ランタイム、token-based pricing、codebase indexing / semantic search / MCP / skills / hooks / subagents が揃い、Rippling / Notion / Faire / C3 AI が SaaS バックエンドへの埋込で先行採用しています。Futurum Group の同日記事は「The editor recedes from the product center, and the Agents Window takes its place」と表現しました。CI/CD ベンダーや Cloud dev environment と競合する位置に移ったと整理する見方もあり、開発インフラとしての評価軸へ移行しつつあると見ています。

### GitHub Copilot Usage-Based Billing ─ コア体験は守り、高負荷だけ実消費に

GitHub Copilot の Usage-Based Billing 移行(2026-04-27 発表 / 2026-06-01 発効)は、エージェント時代に合わせた課金の組み替えです。月額据え置きと同額の AI Credits 付与($10/1,000 / $39/3,900 / $19/1,900 / $39/3,900)、コード補完と Next Edit Suggestions の AI Credits 不消費維持、Annual Plan の廃止、fallback の終了、enterprise / cost center / user の 3 階層予算制御、組織横断 Credits プール、Business / Enterprise 既存への 2026-06-01 〜 09-01 のプロモ Credits($30/$70、+58% / +79% 増し)が一括で並びました。コミュニティ反発は強く、Sonnet 4.6 9x / Opus 27x の Multiplier 議論や Discussion #192948 の 670 downvotes が広く流通していますが、コア体験(補完・NES)を flat-rate で守る設計判断は、移行後の評価にも長く影響すると見ています。

### 永続エージェント基盤の輪郭 ─ Codex CLI の `/goal` と Claude の Memory beta

エージェントが「セッションごとに記憶を失う」段階から、「永続的に目標と記憶を持つ」段階に移りつつあります。Codex CLI 0.128.0(2026-04-30)は Persisted `/goal` workflows(create / pause / resume / clear)、`codex update`、configurable TUI keymaps、plugin marketplace、external agent session import、MultiAgentV2 thread caps を投入しました。Claude 側は Memory for Managed Agents(workspace スコープ、最大 8 stores / session、`memver_...` 30 日保持、`read_write` / `read_only`)が Netflix / Rakuten / Wisedocs / Ando で 97% エラー削減という事例を持ちます。Google AI も Custom MCP server data stores Public Preview と Asynchronous function calling を出しました。三者の出し方は違いますが、向きは共通していると見ています。

### Gemini Enterprise Limited Availability + Volvo Cars 統合 ─ 規制業界と消費者面の同時拡張

Gemini Enterprise で 3.1 Pro / 3 Flash の Limited Availability(2026-04-30、GA SLO + Standard data residency US/EU/global multi-regions)が始まり、規制業界が SLA 契約のもとで本番投入できる準備が整いました。同日、Gemini in Volvo Cars(米国先行 16 モデル / 2020 年以降、trip planning / nav / message dictation / media control)が車内コンパニオン市場へ展開されました。Cloud Next '26 で発表された機能の Public Preview / GA への落とし込みが連続している局面で、次の山は Google I/O 2026(2026-05-19 〜 05-20)になりそうだと見ています。
